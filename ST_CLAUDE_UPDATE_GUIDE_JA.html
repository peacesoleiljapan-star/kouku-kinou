言語聴覚士向け Claude 画面修正手順

言語聴覚士が Claude で index.html を作って管理者へ渡すまでの手順

この資料は、言語聴覚士が Claude を使って新しい index.html を作り、管理者へ渡すまでの手順です。

この手順では、GitHub、PowerShell、server.py の編集はしません。そこは管理者が担当します。

最初に覚えること

  1. 触るのは index.html だけです。
  2. server.py、compose.yaml、Dockerfile などは触りません。
  3. Claude には、差分ではなく完成した index.html 全体を出してもらいます。
  4. 本番へ反映するかどうかの判断は、管理者の検証結果で決まります。

この手順でやること

  1. 今の index.html を Claude に渡します。
  2. 直したい内容を文章で伝えます。
  3. Claude から新しい index.html 全体を受け取ります。
  4. 別名ファイルとして保存します。
  5. 修正内容のメモと一緒に管理者へ渡します。

事前に準備するもの

  1. 現在使っている index.html
  2. 直したい内容のメモ
  3. Claude を開ける PC

直したい内容のメモは、次のように短く箇条書きで十分です。

  1. 利用者情報タブで年齢をもっと見やすくしたい
  2. 総合評価のコメント欄を広くしたい
  3. 印刷時の見出しを分かりやすくしたい

Claude への依頼手順

  1. Claude を開きます。
  2. 今使っている index.html を添付します。
  3. 下の依頼文をそのまま貼ります。
  4. 「直したい内容」の部分だけ、自分の修正内容に置き換えます。
添付した index.html を元に、次の修正を入れた完成版の index.html を全文で返してください。

条件:
- 返答は index.html 全体だけにしてください
- 省略せず、途中を ... にしないでください
- 既存機能は消さないでください
- tab-patient、tab-oral、tab-mna、tab-summary、tab-history は残してください
- showTab(id)、selectMNA()、saveRecord()、deleteRecord(id)、initializeApp() は残してください
- const API_ROOT = '/api/records'; と const SETTINGS_API_ROOT = '/api/settings'; は残してください
- index.html 以外のファイル案は不要です
- ログイン画面だけではなく、評価画面まで含む完成版を返してください

直したい内容:
- ここに修正したい内容を書く

管理者が受け入れる条件

管理者は、受け取った HTML を次のコマンドで検証します。

& "C:\Users\user\AppData\Local\Microsoft\WindowsApps\python.exe" .\server.py --validate-client-html "C:\path\to\index_new.html"

管理者が受け入れるのは次のどちらかが出たときだけです。

  1. Client HTML validation: OK (managed)
  2. Client HTML validation: OK (legacy-source)

次のものは受け入れられません。

  1. index.html の一部分だけ
  2. 途中が省略された HTML
  3. server.py も直す前提の提案
  4. Client HTML validation failed: ... になった HTML

この条件を知っておくと、Claude への依頼文をやり直すときに迷いにくくなります。

Claude の返答を確認するときのポイント

次の 4 つだけ見れば大丈夫です。

  1. index.html 全体が返ってきている
  2. 途中が省略されていない
  3. 自分が頼んだ修正が入っている
  4. server.py など別ファイルの変更提案になっていない

次のような返答は、そのまま使いません。

  1. 差分だけ
  2. 一部分だけ
  3. 「続きは省略します」と書かれているもの
  4. index.html 以外のファイルを直す前提のもの

その場合は Claude に、次の一文を追加で送ってください。

差分ではなく、完成した index.html 全体を省略なしで返してください。

ファイルの保存方法

  1. Claude の返答を全文コピーします。
  2. メモ帳、VS Code、または使いやすいテキストエディタを開きます。
  3. 新しいファイルに全文を貼ります。
  4. 文字コードは通常の UTF-8 のままで保存します。
  5. ファイル名は、たとえば次のようにして保存します。
index_2026-05-20.html

重要:

  1. いきなり元の index.html を上書きしません。
  2. まずは別名で保存して、管理者へ渡します。

管理者へ渡すときに一緒に伝えること

管理者には、ファイルだけでなく、次の 2 つも一緒に渡します。

  1. 何を直したかったか
  2. Claude が直した内容の要約

例:

  1. 利用者情報の見やすさを改善
  2. 総合評価のコメント欄を広げた
  3. 印刷時のタイトルを調整した

やってはいけないこと

  1. index.html の一部分だけを管理者へ渡す
  2. 自分で GitHub の本番ファイルを直接上書きする
  3. server.py を自分で直そうとする
  4. Claude の説明文だけを渡して、HTML ファイル本体を渡さない

うまくいかないとき

Claude が一部分しか返さない

次の一文を追加で送ってください。

省略せず、index.html 全体を最初から最後までそのまま出してください。

Claude が別ファイルの修正を提案する

次の一文を追加で送ってください。

今回は index.html だけを修正対象にしてください。ほかのファイル案は不要です。

自分で正しいか判断できない

無理に直さず、そのまま管理者へ渡してください。最終確認は管理者が行います。

このあと管理者がやること

この資料のここまでできたら、あなたの作業は終わりです。

このあと管理者が次を行います。

  1. PowerShell で検証する
  2. 問題なければリポジトリの index.html と差し替える
  3. GitHub へ反映する
  4. GitHub Actions と本番画面で確認する

ここだけ覚えれば大丈夫: あなたは Claude で完成した index.html を作り、別名で保存して、修正内容のメモと一緒に管理者へ渡せば完了です。
